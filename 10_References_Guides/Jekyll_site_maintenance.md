# Jekyll Site Maintenance Guide

This repository is the source for a GitHub Pages site built with Jekyll. The Markdown files are not served directly as Markdown on the final website. GitHub Pages runs a build step that converts the source files into a static HTML site.

## How the site is built

The main configuration file is `_config.yml`.

At build time, Jekyll:

1. Reads `_config.yml`.
2. Loads the remote Cayman theme from `remote_theme`.
3. Converts Markdown files such as `00_project_overview/index.md` into HTML pages.
4. Wraps each page with the layout in `_layouts/custom.html`.
5. Processes Sass from `assets/css/style.scss` into CSS.
6. Copies static assets such as JavaScript from `assets/js/`.
7. Serves the generated site from an output folder called `_site`.

On GitHub Pages, the `_site` folder is created on GitHub's servers, so it normally does not appear in the repository. Do not manually commit generated HTML files unless the deployment method changes.

## Important Files

### `_config.yml`

Use `_config.yml` for site-wide metadata and folder defaults.

Edit this file when:

- Adding a new major section.
- Adding breadcrumb labels for a section or subsection.
- Changing the active theme.
- Setting `baseurl` for a project GitHub Pages site.

Example section default:

```yml
  - scope:
      path: "03_New_Section"
    values:
      section_title: "New Section"
      section_url: "/03_New_Section/"
```

Only add `section_url` when the folder has an `index.md`.

Example subsection default:

```yml
  - scope:
      path: "03_New_Section/derivations"
    values:
      section_title: "New Section"
      section_url: "/03_New_Section/"
      subsection_title: "Derivations"
```

If that subsection later gets its own `index.md`, add:

```yml
      subsection_url: "/03_New_Section/derivations/"
```

### `_layouts/custom.html`

This is the wrapper used by all pages by default. It currently:

- Adds the shared page navigation.
- Outputs the page content with `{% raw %}{{ content }}{% endraw %}`.
- Loads the MathJax configuration.
- Loads MathJax.
- Loads Mermaid support.

Edit this file when changing page-wide behavior that should appear on every page.

Do not edit this file just to change one page. Put one-page content in that page's Markdown file instead.

### `_includes/page-nav.html`

This file renders the breadcrumb navigation:

```text
Home / Section / Subsection
```

It reads values from each page, usually supplied by `_config.yml` defaults:

- `section_title`
- `section_url`
- `subsection_title`
- `subsection_url`

Edit this file when changing the breadcrumb structure or wording logic for every page.

### `assets/css/style.scss`

This is the main custom stylesheet.

The top of the file has:

```scss
---
---
@import "{% raw %}{{ site.theme }}{% endraw %}";
```

The front matter tells Jekyll to process the file. The `@import "{% raw %}{{ site.theme }}{% endraw %}"` line imports the remote theme's CSS through Jekyll/Liquid. VS Code may highlight this line as unusual SCSS, but it is expected in a Jekyll theme override file.

Edit this file when changing visual styling, including the breadcrumb style.

### `assets/js/mathjax-config.js`

This file holds the website MathJax configuration and custom macros.

Edit this file when adding commands such as:

```js
pdv: ['\\frac{\\partial #1}{\\partial #2}', 2]
```

The local Markdown Preview Enhanced/Crossnote config at `C:/Users/AW/.crossnote/config.js` is separate. GitHub Pages cannot read local files from a contributor's computer, so macros may need to be copied into both places if local preview and deployed rendering should match.

## Adding a New Page

1. Create a Markdown file in the correct folder.
2. Link it from the nearest `index.md`.
3. If the folder already has `_config.yml` defaults, the breadcrumb should appear automatically.
4. If it is a new section or subsection, add a new `_config.yml` default block.

For example:

```text
03_New_Section/index.md
03_New_Section/derivations/example_derivation.md
```

Then add:

```yml
  - scope:
      path: "03_New_Section"
    values:
      section_title: "New Section"
      section_url: "/03_New_Section/"
  - scope:
      path: "03_New_Section/derivations"
    values:
      section_title: "New Section"
      section_url: "/03_New_Section/"
      subsection_title: "Derivations"
```

## Linking Between Pages

Normal Markdown links can point to `.md` files or final `.html` files, but using final site paths is often clearer for GitHub Pages:

```md
[Born-Oppenheimer expansion](../02_Born_Oppenheimer_and_Nonadiabaticity/beginer/01_bornhuang_expansion.html)
```

Inside Mermaid diagrams, use final website paths such as `.html`, because Mermaid links are not converted by Jekyll's Markdown link processor:

```mermaid
click BO "../02_Born_Oppenheimer_and_Nonadiabaticity/beginer/01_bornhuang_expansion.html" "View page"
```

## What Not to Edit

Do not manually edit or commit `_site/`. It is generated output.

Do not add navigation links manually to every Markdown file unless a page needs a special one-off link. Prefer `_config.yml`, `_layouts/custom.html`, and `_includes/page-nav.html` for site-wide behavior.
